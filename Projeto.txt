8. app.py
Função create_app():

Instancia Flask(__name__)
- Carrega configurações com app.config.from_object(Config)
Registra os 5 blueprints: auth_bp, products_bp, cart_bp, orders_bp, admin_bp


@app.before_request — carregar_usuario_atual():

Lê session.get("usuario_id")
Se existir, chama User.buscar_por_id() e atribui a g.usuario_atual
Se não existir, atribui AnonymousUser() a g.usuario_atual


@app.context_processor — injetar_contexto_global():

Se g.usuario_atual.is_authenticated, chama Cart.contar_itens() para o badge do menu
Retorna dicionário {"current_user": g.usuario_atual, "cart_count": cart_count} — disponível em todos os templates automaticamente


Rota "/" — home():

Chama Product.destaques(8)
Renderiza index.html passando produtos_destaque


Bloco with app.app_context():

Chama init_db() para garantir que as tabelas existam
Checa User.email_existe("admin@pastorao.com.br"), se não existir chama User.criar() com is_admin=True
    - Chama _seed_produtos_demo()


Função _seed_produtos_demo():

Checa Product.contar_total() > 0 e retorna imediatamente se já houver produtos
Define lista de dicionários com produtos de demonstração (um por categoria)
 - Percorre a lista chamando Product.criar() para cada um


Variável app = create_app() no nível do módulo
Bloco if __name__ == "__main__" chamando app.run(debug=True)
9. routes/auth_routes.py
auth_bp = Blueprint("auth", __name__)
Rota "/cadastro" GET/POST:

GET: renderiza auth/register.html
- POST: lê nome, email, telefone, senha, confirmar_senha de request.form; chama registrar_usuario(); em caso de sucesso salva session["usuario_id"], chama flash() e redireciona para "home"; em caso de AuthError chama flash() com a mensagem de erro e re-renderiza o formulário


Rota "/login" GET/POST:
   - GET: renderiza auth/login.html

POST: lê email e senha de request.form; chama autenticar_usuario(); em caso de sucesso salva session["usuario_id"] e redireciona para "admin.dashboard" se usuario.is_admin ou "home" se não for; em caso de AuthError re-renderiza com flash


Rota "/logout":

Remove "usuario_id" da sessão com session.pop()
Chama flash() e redireciona para "home"

11. routes/cart_routes.py
cart_bp = Blueprint("cart", __name__)
Função _login_obrigatorio():

Checa g.usuario_atual.is_authenticated
Se não estiver logado, chama flash() e retorna redirect(url_for("auth.login"))
Se estiver logado, retorna None — as rotas checam o retorno antes de continuar


Rota "/carrinho" GET:
 - Chama _login_obrigatorio()

Chama obter_resumo(g.usuario_atual.id)
Renderiza cart/cart.html passando itens_carrinho, subtotal, taxa_entrega, total


Rota "/carrinho/adicionar" POST:

Chama _login_obrigatorio()
Lê produto_id e quantidade de request.form
Chama adicionar_ao_carrinho(), captura CartError com flash
Redireciona para "cart.ver_carrinho"


Rota "/carrinho/atualizar" POST:

Chama _login_obrigatorio()
Lê item_id e quantidade de request.form
Chama atualizar_item(), captura CartError com flash
Redireciona para "cart.ver_carrinho"


Rota "/carrinho/remover" POST:

Chama _login_obrigatorio()
Lê item_id de request.form
Chama remover_item()
Flash de confirmação e redireciona para "cart.ver_carrinho"


Rota "/checkout" GET/POST:

Chama _login_obrigatorio()
Lê endereco_entrega de request.form se for POST, senão None
Chama finalizar_compra(), captura OrderError redirecionando para o carrinho com flash
Em caso de sucesso redireciona para "orders.detalhes" passando o pedido_id

13. routes/admin_routes.py
admin_bp = Blueprint("admin", __name__, url_prefix="/admin")
@admin_bp.before_request — exigir_admin():

Se não estiver logado: flash e redireciona para login
   - Se estiver logado mas is_admin for False: abort(403)
Esse decorator protege todas as rotas do blueprint de uma vez


Rota "/dashboard":

Monta dicionário estatisticas com Order.vendas_de_hoje(), Order.contar_pendentes(), Product.contar_total(), Product.contar_estoque_baixo()
Chama Order.listar_recentes(5)
Renderiza admin/dashboard.html


Rota "/produtos":

Chama Product.listar() sem filtros
Renderiza admin/products.html


Rota "/produtos/novo" GET/POST:

GET: renderiza products/add_product.html passando produto=None — o template usa isso para saber que está no modo cadastro
POST: chama _dados_produto_do_form(), depois Product.criar(), flash e redireciona


Rota "/produtos/<int:produto_id>/editar" GET/POST:

Busca o produto, abort(404) se não existir
GET: renderiza products/add_product.html passando o produto — template preenche os campos automaticamente
    - POST: chama _dados_produto_do_form(), depois Product.atualizar(), flash e redireciona


Rota "/produtos/<int:produto_id>/excluir" POST:

Chama Product.excluir(), flash e redireciona para "admin.produtos"


Rota "/pedidos":
    - Lê status de request.args.get() — filtro opcional
   - Chama Order.listar_todos(status=status)
- Renderiza admin/orders.html
Rota "/pedidos/<int:pedido_id>/status" POST:

Lê status de request.form
Chama atualizar_status_pedido(), captura OrderError com flash
Redireciona para "admin.pedidos"


Função _dados_produto_do_form():
    - Lê todos os campos do formulário de produto com request.form.get()
- Converte preco para float, estoque e desconto para int

Trata campos opcionais (preco_antigo, peso, imagem, etc.) retornando None se vazios
destaque vira bool checando se o campo checkbox veio no form
Retorna dicionário pronto para passar ao Product.criar() ou Product.atualizar()
