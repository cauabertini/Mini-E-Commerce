4- Criar o models:
    Ordem:
        User -> Product -> Cart -> Order
    User:
        Criar a classe user, funcao __init__
User (models/user.py):

Classe User com __init__ recebendo uma row do banco
Property is_authenticated retornando True
Método check_password(senha) usando check_password_hash
Método estático criar(nome, email, senha, telefone, is_admin) gerando hash com generate_password_hash antes de salvar
Método estático buscar_por_id(usuario_id)
Método estático buscar_por_email(email)
Método estático email_existe(email)
Classe AnonymousUser com id=None, is_admin=False, is_authenticated=False


5. models/product.py:

Classe Product só com métodos estáticos
listar(categoria, busca) montando query dinâmica com WHERE 1=1
destaques(limite) filtrando por destaque = 1
buscar_por_id(produto_id)
relacionados(categoria, excluir_id, limite) com ORDER BY RANDOM()
criar(dados) recebendo dicionário
atualizar(produto_id, dados)
excluir(produto_id)
baixar_estoque(produto_id, quantidade) com UPDATE ... SET estoque = estoque - ?
contar_estoque_baixo(limite) para o dashboard
contar_total() para o dashboard


6. models/cart.py:

Classe Cart só com métodos estáticos
listar_itens(usuario_id) buscando cada produto via Product.buscar_por_id e montando lista de dicionários com {id, produto_id, quantidade, produto}
contar_itens(usuario_id) com SUM(quantidade) para o badge do menu
buscar_item(item_id, usuario_id) filtrando pelos dois ids para segurança
adicionar(usuario_id, produto_id, quantidade) verificando se já existe e somando quantidade se sim
atualizar_quantidade(item_id, usuario_id, quantidade) chamando remover se quantidade <= 0
remover(item_id, usuario_id)
limpar(usuario_id) deletando todos os itens do usuário
calcular_totais(itens, taxa_entrega, frete_gratis_acima_de) retornando tupla (subtotal, entrega, total)


7. models/order.py:

Função auxiliar _parse_data(valor) convertendo string do SQLite em datetime
Classe Order só com métodos estáticos
Método privado _montar(pedido_row, com_cliente) buscando pedido_itens, o produto de cada item e opcionalmente o cliente, retornando dicionário completo
criar(usuario_id, itens_carrinho, subtotal, taxa_entrega, total, endereco) inserindo em pedidos, depois em pedido_itens congelando o preço, depois chamando Product.baixar_estoque
listar_por_usuario(usuario_id) ordenado por data DESC
listar_todos(status) com filtro opcional, incluindo cliente
listar_recentes(limite) para o dashboard
buscar_por_id(pedido_id, usuario_id) com usuario_id opcional para segurança
atualizar_status(pedido_id, status)
cancelar(pedido_id, usuario_id)
contar_pendentes() para o dashboard
vendas_de_hoje() ignorando cancelados


8. services/auth_services.py:

Classe AuthError(Exception)
registrar_usuario(nome, email, senha, confirmar_senha, telefone) validando campos obrigatórios, tamanho mínimo da senha, senhas iguais e email duplicado, depois chamando User.criar
autenticar_usuario(email, senha) buscando por email e verificando hash, com mensagem de erro genérica para não vazar se o email existe


9. services/cart_services.py:

Classe CartError(Exception)
adicionar_ao_carrinho(usuario_id, produto_id, quantidade) verificando se produto existe e tem estoque
atualizar_item(item_id, usuario_id, quantidade) verificando se o item pertence ao usuário
remover_item(item_id, usuario_id)
obter_resumo(usuario_id) lendo TAXA_ENTREGA e FRETE_GRATIS_ACIMA_DE do current_app.config e retornando (itens, subtotal, entrega, total)


10. services/order_services.py:

Classe OrderError(Exception)
finalizar_compra(usuario_id, endereco_entrega) verificando carrinho não vazio, estoque suficiente de cada item, calculando totais, chamando Order.criar e depois Cart.limpar
cancelar_pedido(pedido_id, usuario_id) bloqueando se status for entregue ou cancelado
atualizar_status_pedido(pedido_id, status) validando contra conjunto de status válidos antes de salvar


11. app.py:

Função create_app() instanciando Flask, carregando Config, registrando os 5 blueprints
before_request carregando g.usuario_atual da sessão (ou AnonymousUser)
context_processor injetando current_user e cart_count em todos os templates
Rota / chamando Product.destaques(8) e renderizando index.html
Bloco with app.app_context() chamando init_db(), criando admin padrão se não existir e chamando _seed_produtos_demo()
Função _seed_produtos_demo() verificando Product.contar_total() > 0 antes de inserir


12. routes/auth_routes.py:

Blueprint auth_bp
Rota /cadastro GET/POST — no POST chama registrar_usuario, salva session["usuario_id"] e redireciona
Rota /login GET/POST — no POST chama autenticar_usuario, salva sessão e redireciona para dashboard se admin ou home se cliente
Rota /logout limpando session e redirecionando


13. routes/products_routes.py:

Blueprint products_bp
Rota /produtos lendo categoria e busca de request.args
Rota /produtos/<int:produto_id> com abort(404) se não encontrar, passando produtos_relacionados


14. routes/cart_routes.py:

Blueprint cart_bp
Função auxiliar _login_obrigatorio() checando g.usuario_atual.is_authenticated
Rota /carrinho GET
Rota /carrinho/adicionar POST
Rota /carrinho/atualizar POST
Rota /carrinho/remover POST
Rota /checkout GET/POST — finaliza a compra e redireciona para detalhes do pedido


15. routes/order_routes.py:

Blueprint orders_bp
Função auxiliar _login_obrigatorio()
Rota /pedidos listando por usuário
Rota /pedidos/<int:pedido_id> com abort(404) se não encontrar ou não pertencer ao usuário
Rota /pedidos/<int:pedido_id>/cancelar POST


16. routes/admin_routes.py:

Blueprint admin_bp com url_prefix="/admin"
before_request verificando is_authenticated e is_admin, com abort(403) se não for admin
Rota /dashboard
Rotas /produtos, /produtos/novo, /produtos/<id>/editar, /produtos/<id>/excluir
Rotas /pedidos, /pedidos/<id>/status
Função auxiliar _dados_produto_do_form() fazendo as conversões de tipo dos campos do formulário


17. templates/ — nessa ordem:

base.html com header, nav, flash messages, bloco content e footer
index.html com hero, categorias e vitrine de destaques
auth/login.html e auth/register.html
products/products.html, products/products_details.html, products/add_product.html (modo cadastro e edição no mesmo arquivo)
cart/cart.html
orders/orders.html e orders/orders_details.html
admin/dashboard.html, admin/products.html, admin/orders.html


18. static/css/style.css e static/js/script.js

CSS: variáveis de cor, reset, componentes (cards, botões, tabelas, badges de status, stepper de quantidade)
JS: toggle do menu mobile, confirmação antes de deletar (data-confirm), busca em tempo real nas tabelas do admin, stepper de quantidade (qty-plus/qty-minus)
